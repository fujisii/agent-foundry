# アルゴリズムレビュー観点

## 問題分類

レビュー開始前に以下のカテゴリに分類する。複数該当する場合はすべて列挙する。

| カテゴリ | 代表的なアルゴリズム・手法 |
|---------|--------------------------|
| DP（動的計画法） | メモ化再帰、ボトムアップDP、区間DP、桁DP |
| グラフ | BFS, DFS, ダイクストラ, ベルマンフォード, ワーシャルフロイド, トポロジカルソート, Union-Find |
| 貪欲法 | 局所最適を連ねて全体最適を狙う手法 |
| 探索 | 二分探索、全探索、バックトラック、分枝限定 |
| 数学 | 素数判定, GCD/LCM, 組み合わせ, 座標幾何, 数論 |
| 文字列 | KMP, Z-algorithm, ローリングハッシュ, トライ木 |
| データ構造 | スタック、キュー、ヒープ、セグメント木、BIT |
| 分割統治 | マージソート, 高速冪乗, FFT |
| シミュレーション | 問題の手順をそのままトレース |

## 解法レビュー

- アルゴリズムとして正しいか
- 典型ケース・境界ケースで破綻しないか
- 前提条件に無理がないか

丸暗記ではなく「なぜその解法か」の理由を重視する。

## 計算量レビュー

- 時間計算量を O(…) で明示し、導出根拠を示す
- 空間計算量を O(…) で明示し、導出根拠を示す
- より効率の良い解法がないか検討する

## 代替解法の比較

改善余地がある場合は以下の形式で比較する。

| 解法 | 時間 | 空間 | メリット | デメリット |
|------|------|------|---------|-----------|

改善がない場合は「改善余地なし」と明記する。

## 注意点・落とし穴

以下の観点で確認する。

- オフバイワン
- 境界条件
- 重複処理
- 再帰の深さ
- データ構造選択ミス
- 前処理の見落とし
