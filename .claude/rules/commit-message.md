# コミットメッセージ規約

## フォーマット

```
type: 変更内容（50文字以内）
```

## type一覧

| type | 用途 |
|---|---|
| `feat` | 新機能の追加 |
| `fix` | バグ修正 |
| `docs` | ドキュメントの変更 |
| `refactor` | 動作変更を伴わない構造改善 |
| `test` | テストの追加・修正 |
| `chore` | ビルド・設定・その他 |

## ルール

- メッセージは日本語で書く
- 動詞で終める：「追加」「修正」「削除」「更新」「リファクタリング」
- 1行で完結させる（本文・フッター不要）
- 変更の「何を」ではなく「なぜ」が非自明な場合は type の選択で表現する

## 例

```
feat: ユーザー認証を追加
fix: ログイン時のバリデーションエラーを修正
refactor: 認証処理を別モジュールに分離
chore: .gitignoreを追加
docs: READMEにセットアップ手順を追加
test: ログイン処理のユニットテストを追加
```
