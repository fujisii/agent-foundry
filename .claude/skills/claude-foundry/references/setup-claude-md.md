# CLAUDE.md セットアップガイド

## CLAUDE.md とは

セッション開始時に自動ロードされる指示ファイル。Claudeが**常に知っておくべき**プロジェクト情報を書く。コードを読めば分かることは書かない（コメントに書けばいい）。コードから読み取れない「なぜ」「ルール」「落とし穴」を書く。

## スコープ（配置場所）

| 場所 | 対象 |
|---|---|
| `./CLAUDE.md` | このプロジェクト（git管理、チーム共有） |
| `./.claude/CLAUDE.md` | 同上（隠しディレクトリ版） |
| `~/.claude/CLAUDE.md` | 全プロジェクト共通（個人設定） |
| `./CLAUDE.local.md` | このプロジェクト・個人専用（gitignore推奨） |

## 品質基準（何を書くべきか）

| カテゴリ | 例 |
|---|---|
| コマンド | `npm run dev`, `npm test`, デプロイ手順 |
| アーキテクチャ | モジュール構成、主要なデータフロー |
| 非自明なルール | 「このAPIは冪等でないため再試行禁止」など |
| 落とし穴 | 「ORMのN+1問題あり、eager loadingを使うこと」 |
| インポート構文 | `@path/to/shared-rules.md` で他ファイルを取り込む |

**書かなくていいもの**: コード自体の説明、明らかなこと、Git履歴で分かること

## ファイル構造テンプレート

```markdown
# プロジェクト名

## コマンド
- ビルド: `npm run build`
- テスト: `npm test`
- 開発サーバー: `npm run dev`
- リント: `npm run lint`

## アーキテクチャ
（プロジェクトの主要な構造を2〜5行で）

## コーディング規約
- （チーム固有のルール）
- （標準から外れる箇所）

## 注意事項
- （よくある落とし穴）
- （非自明な制約）
```

## インポート構文

`@`プレフィックスで外部ファイルをインポートできる（最大5段階）。

```markdown
# CLAUDE.md
@.claude/rules/api-conventions.md
@.claude/rules/testing-conventions.md
```

## 確認すべき質問

1. どのコマンドをClaudeが知っておくべきですか？（ビルド・テスト・デプロイ等）
2. コードを見ても分からない非自明なルールや落とし穴はありますか？
3. モノレポですか？（複数サブプロジェクトがある場合、importで分割する）

## ベストプラクティス

- **200行以内**を目標にする（長いとコンテキストを消費する）
- 長くなる場合は `@import` でファイルを分割する
- 定期的に見直して古くなった情報を削除する
- HTMLコメント（`<!-- -->`）はClaudeには見えない（メモに使える）

## 作成後のアクション

- 新しいセッションを開始すると自動ロードされる
- `/memory` コマンドでロードされているか確認できる
- `@` importの確認は `claude --debug` で行える
