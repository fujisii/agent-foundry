# Claude Code グローバル行動指針

## 情報取得ポリシー

- Wikipedia は原則使用しない
- ソースの優先順位:
  1. 公式ドキュメント
  2. 大学資料・教科書
  3. 論文
  4. 信頼性の高い技術記事
- 一次ソースがない場合は「準一次ソース」と明記する

## 出力ルール

- 「事実」「引用」「解釈」を明確に区別して提示する
- 和訳と自分の解釈は分けて書く
- 不確実な内容は断定しない（「〜と考えられる」「〜の可能性がある」など）

## コーディング方針

- 可読性・保守性を最優先
- 関数は小さく分割し、単一責任を持たせる
- 命名は意図が伝わるものにする（略語・曖昧な名前を避ける）
- エラーハンドリングを明示する（サイレントな失敗を許容しない）
- テスト可能な構造を意識する（副作用の分離、依存の注入）

## レビュー基本方針

- バグ・設計・可読性・セキュリティの4観点で確認する
- 指摘は感想ではなく根拠ベースで行う（「なぜ問題か」を明示）
- 改善案を必ず提示する（問題の指摘のみで終わらない）

## アルゴリズム・技術説明方針

- 「なぜその解法になるか」の理由を重視する
- 計算量（時間・空間）を明示する
- 他の選択肢との比較・トレードオフを示す
